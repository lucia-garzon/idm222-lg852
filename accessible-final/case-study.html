Accessible Portfolio

Overview

This was a solo based project which involved development of my professional portfolio, built specifically for my Scripting for Accessibility class. The site includes a home page with project cards, a resume page, and an "About" page that features a fully functional and accessible contact form. My main focus was making sure that someone using assistive technology would have the same high-quality experience as any other visitor, ensuring that my design didn't just look good but was also "Robust", or working across different browsers and assistive technologies. The project was a chance for me to apply everything I learned about Inclusive and Ethical design. Instead of just checking boxes, I wanted to build a site that feels intuitive for someone using a screen reader or keyboard. I focused heavily on manual testing to make sure my "Skip to main content" links and ARIA landmarks actually improved the user experience. By the end of the project, I had a responsive, accessible site that meets the high standards for inclusive design we studied throughout the term.

The Context and Challenge

The project began as the final requirement for IDM380, where we spent the term learning how to bridge the gap between standard web development and truly accessible design. My goal was to take the three core pages of my portfolio and ensure they weren't just pretty to look at, but functionally sound for everyone. I worked on this as a solo project, handling everything from the initial HTML structure to the final CSS styling.

The core problem I wanted to solve was "exclusionary design." So many portfolios use non-semantic tags and fixed pixel sizes that break when a user tries to customize their viewing experience. I wanted to prove that a professional, modern aesthetic could coexist with strict accessibility standards.

One example was moving away from "div-based" coding and focusing on semantic structure. Through code, I had to add more labels and consistent structure to all aspects of the site, assuming that anyone and everyone wants to be able to read it.

I measured my success based on how the site responded during manual testing. This meant every interactive element had to be reachable and understandable for a user who might not be using a mouse. In addition, I had to be considerate of special use cases by providing options for reduced motion, allowing for responsive typography when the user zooms in, and providing dark mode based on the user's preference.

The Process and Insight

My process began with selecting the right foundation for my typography. I chose the Atkinson Hyperlegible font family, which was specifically designed to increase character recognition and improve legibility for users with low vision. I implemented this using a variable font file to keep the site performing quickly while maintaining high readability.

Next, I made sure to use relative units like rem for all my spacing and sizing. This ensures that if a user changes their default browser font size, my layout scales gracefully without breaking. I also coded a "Skip Navigation" link right at the top of every page so keyboard users can jump straight to the content without having to tab through the menu every time.

The most valuable part of my process was testing with VoiceOver. By listening to my site, I realized where I needed to add specific aria-labels to clarify things. For example, I added labels to my social media links in the footer so a screen reader would announce "LinkedIn icon" or "Github icon" instead of just a generic link. It taught me that good UX is as much about what you hear as what you see.

The Solution

The final solution is a streamlined, three-page portfolio that prioritizes the user experience. On the home page, I used a clear heading hierarchy and clear semantic tags so that someone using a screen reader can easily scan through my different project cards. I also made sure my "Lilypad" logo was clearly labeled so the brand identity is accessible to everyone.

For the contact form on the "About" page, I focused on making the interaction as smooth as possible. I used the aria-invalid attribute and custom error messages in my JavaScript to let users know exactly which field needs a fix. I also included a "Discard" button that resets the form and provides clear feedback that the action was successful.

The resume page was designed to be much more than a static document. I used a clear heading structure so that someone using a screen reader can quickly jump between sections like "Experience" and "Education" without having to listen to the entire page. I also included a "Skip to top" link in the footer of every page, which helps users quickly return to the navigation menu without excessive scrolling.

The Results

This project was a success because it passed my most rigorous test: a full site walkthrough using only VoiceOver and my keyboard. I was able to navigate the case study cards, fill out my contact form, and read through my resume without a single hurdle. The focus states are clearly visible throughout the site, so a keyboard user always knows exactly where they are.

The project was a huge success because it proved that you don't have to sacrifice style for accessibility. By the final build, I was able to navigate the entire site using only the "Tab" and "Enter" keys while listening to VoiceOver. Everything was announced correctly, and the focus indicators were always visible so I never lost track of where I was on the page.

Looking back, the biggest lesson I learned was how much small details matter, like using the right ARIA roles for dynamic content. If I were to change anything in the future, I might experiment even more with advanced CSS techniques to handle high-contrast modes. Overall, this project gave me a solid foundation in building web experiences that are inclusive of everyone.

Moving forward, I feel much more confident in my ability to write "clean" code that respects the user. I’ve learned that accessibility isn't a feature you add at the end—it’s a mindset you start with. This portfolio is now a solid foundation that I can continue to build on as my career grows.

UI/UX Designer based in Philadelphia, PA
